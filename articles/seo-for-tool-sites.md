ブログやメディアのSEOノウハウをそのままツールサイトに適用しても、期待通りの結果が出ないことが多い。[AND TOOLS](https://and-tools.net/) を2年以上運営してきて、ツールサイト固有のSEO戦略があることを痛感した。記事サイトとの違いを整理しながら、ツールサイトで効果的だった施策をまとめる。

## ツールサイトとコンテンツサイトの根本的な違い

SEOの観点での最大の違いは**検索意図の種類**だ。

| 検索意図 | 例 | 向いているコンテンツ |
|---------|-----|---------------------|
| Know（知りたい） | 「消費税とは」 | ブログ記事 |
| Do（やりたい） | 「消費税 計算」 | ツール |
| Go（行きたい） | 「AND TOOLS」 | ブランドページ |
| Buy（買いたい） | 「会計ソフト おすすめ」 | 比較記事 |

ツールサイトは「Do系キーワード」に圧倒的な強みを持つ。「〇〇 計算」「〇〇 換算」「〇〇 チェック」という検索に対して、ツール自体が最高の回答になるからだ。

### 記事を書かなくていいのか

「ツールを作れば記事はいらない」は間違いだ。ツールページだけだとカバーできる検索意図が限られる。

[AND TOOLS](https://and-tools.net/) では各ツールに解説コンテンツを付けている。例えば消費税計算ツールのページには「軽減税率の対象品目一覧」「インボイス制度との関係」を説明する文章も含める。これにより：

- Know系キーワードもカバーできる
- ページの文字数が増えてコンテンツとして評価されやすくなる
- 関連する検索意図を1ページで捕捉できる

## ツールサイト特有のSEO施策

### 1. ツール名を正確にキーワードとして使う

記事サイトでは「検索ボリュームのあるキーワードで記事タイトルを作る」が基本だ。ツールサイトでも同じ発想が重要だが、キーワードの粒度が変わる。

良い例：

```
タイトル: 消費税計算ツール — 税込・税抜を瞬時に計算
URL: /tool/consumption-tax/
H1: 消費税計算ツール（税込・税抜）
```

避けるべき例：

```
タイトル: 消費税を計算するためのページ
URL: /tool/tax/
```

URLにもキーワードを入れる。ツール名はユーザーが検索するそのままの言葉を使う。「消費税計算ツール」「消費税 計算」の両方を意識してタイトルとH1を設計する。

### 2. ツール数と検索インデックスのバランス

ツールを大量に作ると問題が起きることがある。Googlebotのクロールバジェットが分散して、重要なページのインデックス登録が遅れるケースだ。

対策として有効なのが：

**sitemap.xmlの優先度設定**

```xml
<url>
  <loc>https://and-tools.net/tool/consumption-tax/</loc>
  <priority>0.9</priority>
  <changefreq>monthly</changefreq>
</url>
<url>
  <loc>https://and-tools.net/tool/unit-convert/</loc>
  <priority>0.5</priority>
  <changefreq>yearly</changefreq>
</url>
```

流入が多いツールは優先度を高く設定する。ただしGoogleはpriority属性を参考程度にしか見ないので、過信しないこと。

より効果的なのが**内部リンクの集中**だ。重要なツールへの内部リンク数を増やすことで、クロールの優先度を間接的にコントロールできる。

### 3. ページの読み込み速度がランキングに直結する

ツールサイトはJavaScriptを多用するため、Core Web Vitalsが悪化しやすい。Googleは2021年からページエクスペリエンスシグナルをランキング要素に加えた。

AND TOOLSで実施してスコアが改善した施策：

**LCP（Largest Contentful Paint）対策**

- ファーストビューに大きな画像を置かない
- ヒーロー画像は WebP に変換してサイズを削減
- Critical CSSをインライン化して描画ブロックを減らす

**CLS（Cumulative Layout Shift）対策**

- AdSenseの広告枠のサイズを事前に確保する
- `min-height` で広告スペースを予約する

```css
.ad-slot {
  min-height: 250px; /* 広告ロード前に高さを確保 */
}
```

**FID/INP（インタラクション応答性）対策**

- 計算処理をメインスレッドから分離（Web Workers）
- イベントハンドラをdebounceする

[ToolShare Lab](https://and-and.net/) はGulp + 静的HTMLで構築しているため、Next.jsのようなSSRフレームワークよりも素のHTMLに近い。JavaScriptの読み込みを最小限にできるのが静的サイトの強みだ。

### 4. 構造化データでリッチリザルトを狙う

ツールサイトでJSON-LDとして有効な構造化データは：

**SoftwareApplication**

```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "消費税計算ツール",
  "applicationCategory": "BusinessApplication",
  "operatingSystem": "Web",
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "JPY"
  },
  "description": "消費税（10%・8%）の税込・税抜を瞬時に計算できる無料ツール"
}
```

**FAQPage**

ツールの使い方に関するFAQを構造化データとして実装すると、検索結果でアコーディオン形式のリッチリザルトが表示される可能性がある。

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "消費税の計算方法は？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "税抜価格 × 1.1（標準税率）または × 1.08（軽減税率）で計算します。"
      }
    }
  ]
}
```

## キーワード選定：ツールサイト版

### ロングテール狙いが基本

「消費税 計算」は月間検索ボリュームが多いが、競合も強い。ツールサイトではもう少し具体的なキーワードを狙う方が現実的だ。

具体的な選定基準：

- **「〇〇 計算 サイト」「〇〇 計算ツール」系**: Do意図が明確
- **業種や状況を限定したキーワード**: 「フリーランス 消費税 計算」「個人事業主 国民健康保険 計算」
- **競合チェック**: 上位10件が大手メディアだけなら避ける。個人サイトが混じっていれば狙い目

Search Consoleで「表示回数は多いが順位が11〜20位のクエリ」を探すのも効果的だ。もう少し最適化すれば1ページ目に上がるページを特定して重点的に改善できる。

### 検索意図のズレに注意

「住宅ローン 計算」で検索するユーザーの一部は計算ツールを求めているが、一部は「住宅ローンの仕組みを知りたい」という人もいる。両方の意図をカバーするためにツール + 解説の組み合わせが有効だ。

## E-E-A-Tとツールサイト

E-E-A-T（Experience, Expertise, Authoritativeness, Trustworthiness）はツールサイトでも重要だ。

**Experience（経験）**: 実際に使ったことが伝わるコンテンツ。「このツールを作った背景」「利用者の声」を載せる。

**Expertise（専門性）**: ツールの計算式の根拠を示す。消費税計算なら消費税法の条文番号を引く。

**Authoritativeness（権威性）**: 被リンクが重要。他の記事やサービスから参照されることで権威性が上がる。

**Trustworthiness（信頼性）**: 運営者情報を明確にする。AND TOOLSでは「このサイトについて」ページに運営者情報・連絡先を記載している。

## 競合分析の方法

ツールサイトの競合は記事サイトとは異なる。以下の観点でチェックする：

1. **Google検索で「〇〇 計算」を検索して上位のサイトを確認**
2. **そのサイトのツール数、UI、解説コンテンツを調査**
3. **PageSpeed Insightsでスコアを比較**
4. **被リンク数をahrefs等で確認**

AND TOOLSで上位を取れているキーワードに共通するのは「大手の計算ページよりUIが使いやすい」か「ニッチすぎて大手が参入していない」のどちらかだ。

## まとめ：ツールサイトSEOの優先施策

1. **Do系キーワードに特化**: 「〇〇 計算」「〇〇 換算」で上位を狙う
2. **ツール + 解説のセット**: ツールだけのページにしない
3. **Core Web Vitals**: LCP・CLS・INPを常に監視して改善する
4. **構造化データ**: SoftwareApplication + FAQを実装する
5. **内部リンク**: 関連ツールへのリンクでPVを伸ばす
6. **Search Consoleで11〜20位のクエリを定期的にチェック**する

記事サイトのSEOノウハウをそのまま流用しようとすると遠回りになる。ツールサイト特有の戦略を意識して取り組むことが、効率的な検索流入増加につながる。
