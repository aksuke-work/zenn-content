Google Search Console（以下GSC）を開いて「表示回数が多いのにクリックが少ない」「順位が上がったのにトラフィックが増えない」と感じたことがあるなら、このまま読んでほしい。[AND TOOLS](https://and-tools.net/) の運営でGSCのデータを毎週確認するようになって、数字の読み方が分かってきた。

## GSCの4つの指標を正しく理解する

### 表示回数（Impressions）

検索結果にURLが表示された回数だ。ユーザーがクリックしなくてもカウントされる。重要なのは「**表示**」の定義で：

- デスクトップ: 画面に表示されればカウント
- モバイル: 画面に表示されればカウント（スクロールして見えなくてもカウントされる場合がある）

「表示回数が急増した」は良いことのように見えるが、原因を確認する必要がある。

**良い表示回数増加:**
- 新しいキーワードで上位表示され始めた
- サイトの被リンクが増えてインデックスが広がった

**悪い表示回数増加:**
- スパムサイトの被リンクでGoogleが誤認識している
- ボットがサイトを巡回している（→ 実際のクリックは増えない）

### クリック数（Clicks）

これがGSCで最も重要な指標だ。実際にページを開いたユーザー数に近い（内部ページへの移動は含まない）。

### 平均クリック率（CTR）

```
CTR = クリック数 ÷ 表示回数 × 100
```

目安として：

| 検索順位 | 平均CTR |
|---------|---------|
| 1位 | 28〜35% |
| 2位 | 15〜20% |
| 3位 | 10〜15% |
| 4〜10位 | 1〜8% |
| 11位以降 | 1%未満 |

AND TOOLSのあるツールページで、順位が8位から3位に上がったとき CTRが2.1%から11.4%に改善した。5倍の改善だ。順位を上げることの価値がCTRの数字でよく分かる。

### 平均掲載順位（Position）

1〜100の数値で表示される。注意点が2つある：

1. **平均値であること**: 複数のキーワードで表示されているページは、キーワードごとの順位を平均した値が表示される
2. **デバイス・地域で異なること**: モバイルとデスクトップで順位が違う。フィルタリングして確認する

## 表示回数と順位の「おかしな関係」を読む

### パターン1: 表示回数が多いが順位が低い

**状況例:**
- 表示回数: 5,000/月
- 平均順位: 25位
- クリック数: 50/月（CTR 1%）

**意味:** 多くのクエリで11〜30位あたりに表示されている。キーワードは検索されているが、1ページ目に上がれていない。

**対策:**
- Search Consoleでそのページに関連するクエリを確認
- 上位表示を妨げている原因を特定（コンテンツ品質、被リンク不足、技術的SEO）
- 11〜20位に集中しているクエリを洗い出して重点的に改善

### パターン2: 順位が良いがCTRが低い

**状況例:**
- 平均順位: 4位
- CTR: 1.2%（4位の平均は5〜7%程度）

**意味:** 検索結果には表示されているが、クリックされていない。

**原因として多いのは:**

1. **タイトルが魅力的でない**: クリックしたくなるタイトルになっていない
2. **メタディスクリプションが弱い**: 検索結果のスニペットが検索者の興味を引いていない
3. **リッチリザルトが表示されていない**: 競合サイトが FAQやレビュースターを表示していてそちらをクリックされている
4. **検索意図とのミスマッチ**: 「計算したい」ユーザーに対して記事ページが表示されている

AND TOOLSで実際に試した改善例：

**Before:** `消費税計算 | AND TOOLS`
**After:** `消費税計算ツール（無料）— 税込・税抜を瞬時に計算`

タイトルを変えて「無料」「瞬時に」というメリットを明示したところ、CTRが1.8%から4.2%に改善した。

### パターン3: クリックは増えているが直帰率が高い

GSCだけでは分からないが、GA4と組み合わせると見えてくる。「ページを開いたがすぐ離脱」という状況はランディングページとコンテンツのミスマッチが原因が多い。

## GSCの効果的な使い方

### 11〜20位のクエリを定期的に掘り起こす

これが最も費用対効果が高い施策だ。

手順：

1. Search Console → 検索パフォーマンス → クエリ
2. フィルターで「掲載順位 > 10」「掲載順位 < 21」を設定
3. 表示回数順に並べる
4. 上位のクエリが対応するページを確認する
5. コンテンツを改善して1ページ目を目指す

[AND TOOLS](https://and-tools.net/) ではこの施策を月1回実施している。15〜20位にいるページは少しの改善で1ページ目に上がることが多い。

### ページ単位で問題のあるページを特定する

クエリ別の分析ではなく、ページ別に見ることも重要だ。

1. Search Console → 検索パフォーマンス → ページ
2. CTRが低いページを特定
3. そのページをクリックして、どんなクエリで表示されているか確認
4. クエリとページの内容がミスマッチなら修正する

### モバイルとデスクトップを分けて確認する

フィルターでデバイスを切り替えると、モバイルとデスクトップで全く違うデータが見えることがある。

AND TOOLSの場合、計算ツールはモバイル利用が多い。ただしモバイルのCore Web VitalsスコアがデスクトップよりLCPが1秒長かった時期があり、モバイル順位だけ低下していた。デバイス別の確認は必須だ。

## カバレッジレポートで技術的問題を発見する

SEOパフォーマンスレポート以外にも重要な情報がある。

### インデックスカバレッジ

「インデックスに未登録」のページがある場合、その理由を確認する：

| ステータス | 意味 | 対処 |
|-----------|------|------|
| クロール済み（インデックス未登録） | Googleが見たがインデックスに値しないと判断 | コンテンツ品質改善 |
| 送信されサイトマップに掲載（インデックス未登録） | クロールすらされていない | サイトマップ再送信 |
| noindexタグを検出 | noindexが付いている | 意図的でなければ削除 |
| クロールのリクエストを受け付けました | インデックス登録待ち | 待つ |

ツールを新たに追加したときにこのレポートでインデックス状況を確認するのが習慣になった。

### Core Web Vitals

モバイル・デスクトップそれぞれで「良好」「改善が必要」「低速」のURLが分類される。「低速」に分類されているURLを優先的に改善する。

[ToolShare Lab](https://and-and.net/) は静的サイトなのでCore Web VitalsはGood判定を維持しやすいが、AdSenseの動的読み込みでCLSが発生することがあり、広告枠の高さを固定することで対処した。

## データの落とし穴：GSCが信用できないケース

### ブランドクエリによる順位の歪み

サイト名やURL直接入力による検索は順位1位になるが、これが平均順位を引き上げる。ブランドクエリをフィルターで除外して「純粋な検索トラフィック」だけを見ることが大事だ。

### 16ヶ月以上前のデータは確認できない

GSCのデータ保持期間は16ヶ月だ。長期的なトレンドを見るためにGSCのデータをBigQueryに定期エクスポートしておくと良い。

### サンプリングによる誤差

データ量が多い場合、GSCはサンプリングを行う。表示されている数字が厳密な値でない場合がある。傾向を掴む指標として使う。

## まとめ：GSC活用の週次ルーティン

AND TOOLSで毎週実施しているGSCレビューの手順を共有する。

**毎週15分:**
1. 検索パフォーマンスで先週のクリック数・表示回数をチェック
2. 急激な変動があるクエリを確認（Google更新かどうか判断）
3. 11〜20位のクエリで表示回数が多いものをメモ

**月1回30分:**
1. 11〜20位クエリの改善対象ページを選定
2. CTRが低いページのタイトル・メタディスクリプションを見直す
3. カバレッジレポートでインデックス問題がないか確認
4. Core Web Vitalsで「低速」URLを確認

GSCのデータは「現状を映す鏡」だ。数字を正しく読んで、具体的な改善アクションに落とし込むことがトラフィック成長への近道だ。
