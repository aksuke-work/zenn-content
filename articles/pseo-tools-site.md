個人開発の税金計算ツールサイト [AND TOOLS](https://and-tools.net/) で、pSEO（プログラマティックSEO）を使って60ページを自動生成した。公開10日で検索表示514回、6ページが1ページ目にランクインした。やったことを全部書く。

## pSEO（プログラマティックSEO）とは

pSEOは「テンプレート + データ」でSEOページを大量生成する手法。

- **Zapier**: 「○○ vs △△」の比較ページを数千件生成
- **Nomad List**: 都市×条件のフィルタページ
- **Wise**: 通貨ペアごとの為替計算ページ

やっていることは単純で、「検索需要のあるパターンを見つけ、テンプレートに変数を流し込んで個別ページを作る」だけ。スパムとの違いは**各ページがユーザーにとって本当に価値があるかどうか**。

## 「年収○○万円の手取り」60ページを生成した

### 検索需要の発見

Google Search Consoleで [手取り計算ツール](https://and-tools.net/tools/take-home-pay/) の流入キーワードを見たら、こんなクエリが大量にあった:

```
年収300万 手取り
年収400万 手取り
年収500万 手取り いくら
年収600万 手取り 計算
...
```

年収の具体的な金額ごとに検索されている。しかもどの金額でも月間検索ボリュームが1,000〜10,000ある。これをツール1ページで拾い切るのは無理がある。

### テンプレート設計

PHPで変数を埋め込むテンプレートを作った。

```php
<?php
// 年収データ（100万〜2,000万、50万刻み）
$incomes = range(1000000, 20000000, 500000);

foreach ($incomes as $income) {
    $incomeMan = $income / 10000; // 万円表記
    $result = calculateTakeHome($income); // 税金計算エンジン

    // テンプレートに流し込み
    $html = renderTemplate('pseo-takehome.php', [
        'income' => $income,
        'incomeMan' => $incomeMan,
        'incomeTax' => $result['incomeTax'],
        'residentTax' => $result['residentTax'],
        'socialInsurance' => $result['socialInsurance'],
        'takeHome' => $result['takeHome'],
        'takeHomeMonthly' => floor($result['takeHome'] / 12),
        'takeHomeRate' => round($result['takeHome'] / $income * 100, 1),
    ]);

    file_put_contents("output/nenshu-{$incomeMan}man.html", $html);
}
```

各ページの構成:

```
H1: 年収{X}万円の手取り額 — 税金・社会保険料の内訳
├── 手取り額サマリー（結論ファースト）
├── 税金・社会保険料の内訳テーブル
├── 月額の手取り
├── 年収±50万との比較テーブル
├── 手取り率の推移グラフ（年収別）
├── 計算の前提条件
└── CTA → メインのシミュレーターへ
```

### スパムにならないための工夫

pSEOでGoogleにスパム判定されないよう、以下を徹底した:

1. **各ページの計算結果が実際に異なる**（テンプレは同じでも値が全部違う）
2. **前後の年収との比較テーブル**で差分の情報を提供
3. **手取り率の変化**を年収別グラフで視覚化（累進課税の影響が見える）
4. **薄いページにしない**。各ページ1,500文字以上で計算ロジックの解説を含む
5. **canonical URLを適切に設定**（重複コンテンツ対策）

## 内部リンク設計

pSEOで最も重要なのが内部リンク構造。ハブ&スポーク型にした。

```
[AND TOOLS トップ]
    ├── [手取り計算ツール]（ハブ）
    │     ├── 年収100万の手取り（スポーク）
    │     ├── 年収150万の手取り
    │     ├── 年収200万の手取り
    │     │   ...
    │     └── 年収2000万の手取り
    │
    ├── [178万の壁 計算ツール]
    ├── [所得税 計算ツール]
    └── [社会保険料 計算ツール]
```

各pSEOページから:
- **メインツールへのリンク**（「詳しくシミュレーションする」CTA）
- **前後の年収ページへのリンク**（年収300万→250万/350万）
- **関連ツールへのリンク**（[178万の壁](https://and-tools.net/tools/178-wall/)、社会保険料計算等）

これでメインツールにPageRankが集中し、pSEOページ同士もクロールされやすくなる。

## llms.txt の設置

ChatGPTやPerplexityなどのLLMがサイト情報を正しく取得できるよう、[llms.txt](https://and-tools.net/llms.txt) を設置した。

```markdown
# AND TOOLS
> フリーランス・個人事業主向けの無料計算ツール集

## ツール一覧
- [手取り計算シミュレーター](https://and-tools.net/tools/take-home-pay/): 年収から手取りを計算
- [所得税 計算ツール](https://and-tools.net/tools/income-tax/): 累進課税の所得税を計算
- [住民税 計算ツール](https://and-tools.net/tools/resident-tax/): 住民税を計算
...
```

`llms.txt` はLLMがサイトの構造を理解するための標準フォーマット。robots.txtのLLM版と考えるとわかりやすい。AI検索が増える中で、従来のSEOに加えてLLM最適化（LLMO）も重要になってきている。

## 結果: 公開10日の数字

Google Search Consoleの数字（公開10日後）:

| 指標 | 数値 |
|------|------|
| 表示回数 | 514回 |
| クリック数 | 12回 |
| 平均掲載順位 | 18.2位 |
| 1ページ目（10位以内）のページ数 | 6ページ |
| インデックス済みページ数 | 48/60 |

10日で48ページがインデックスされ、6ページが1ページ目に入った。新規ドメインでこの速度は悪くない。

特に効果が高かったキーワード:
- 「年収350万 手取り」→ 8位
- 「年収450万 手取り」→ 6位
- 「年収550万 手取り いくら」→ 9位

ボリュームの大きい「年収500万 手取り」「年収600万 手取り」は競合が強く、まだ2ページ目。ここはドメインパワーが育てば上がるはず。

## 今後の展開

現在AND TOOLSには34個のツールがある。各ツールでpSEOを展開すれば:

- 手取り計算 → 60ページ（済）
- 社会保険料計算 → 都道府県47ページ
- ローン計算 → 金額×金利のマトリクスで100ページ
- 法人化シミュレーション → 事業所得別に30ページ

合計で **200〜300ページ** を目指す。すべて計算結果が異なる実用的なページなので、スパムにはならない。

## pSEOの判断基準

すべてのキーワードパターンでpSEOが有効なわけではない。以下の条件を満たすときに使う:

1. **同じ構造で変数だけ異なる検索需要**がある（「年収○○万」「○○県の社会保険料」）
2. **各ページの内容が実際に異なる**（計算結果が変わる）
3. **ユーザーが具体的な値を求めている**（「年収400万 手取り」は具体値が欲しい）
4. **競合がpSEOをやっていない or やり方が雑**

逆に、変数を変えても内容がほぼ同じになるパターンはやめたほうがいい。Googleの「有用でないコンテンツ」に引っかかる。

## まとめ

pSEOは「プログラマーだからこそ活かせるSEO手法」だと思う。

1. **検索需要のパターンを見つける**（GSCのデータが起点）
2. **テンプレート + 計算エンジン**でページを生成
3. **内部リンク設計**でPageRankをハブに集中
4. **各ページが本当に価値を持つ**ことがスパム回避の条件
5. **llms.txtでLLM最適化**も忘れずに

個人開発のサイトでもPV数千/月は十分狙える。ツール × pSEO × AdSense/アフィリエイトの組み合わせは、エンジニアの副業として相性が良い。
