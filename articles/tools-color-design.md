「このサイトの配色、なんかしっくりこない」「メインカラーは決まっているけど、アクセントカラーが決まらない」——こういう悩みはデザイン経験に関係なく、毎回やってくる。

配色は感覚だけでなく、色彩理論と数値に基づいてアプローチすることで解決できる場合が多い。本稿では、配色の迷いを解消するために実際に使っているツールをまとめる。

## 配色の基礎知識（おさらい）

ツールを使う前に、最低限知っておくべき概念を整理する。

### 色相・彩度・明度

- **色相（Hue）**: 赤・青・緑など、色の種類
- **彩度（Saturation）**: 色の鮮やかさ。0%がグレー、100%が最鮮明
- **明度（Lightness）**: 色の明るさ。0%が黒、100%が白

HSL記法 `hsl(210, 80%, 50%)` はこの3軸で色を定義するため、RGBや16進数より直感的に色を調整しやすい。

### 主要な配色パターン

| パターン | 概要 | 使う場面 |
|---|---|---|
| 補色 | 色相環で180°反対の色 | アクセントを際立たせる |
| 類似色 | 色相環で30°以内の色 | まとまった印象を作る |
| トライアディック | 色相環で120°ずつの3色 | 賑やかで活発な印象 |
| モノクロマティック | 同色相の明度・彩度違い | 洗練されたシンプルな印象 |

## 配色ツール一覧

### 1. カラーパレット生成ツール — ToolShare Lab

**URL**: [https://and-and.net/](https://and-and.net/)
**用途**: 1色から調和したパレットを一括生成

[ToolShare Lab](https://and-and.net/) のカラーパレット生成ツールは、ベースカラーを1色入力するだけで、補色・類似色・トライアディックなどのパターン別に複数のカラーを自動生成する。

特に便利なのは、生成されたカラーコードをCSS変数形式でそのまま出力できる点だ。

```css
:root {
  --color-primary: #3B82F6;
  --color-primary-light: #93C5FD;
  --color-primary-dark: #1D4ED8;
  --color-accent: #F59E0B;
}
```

このまま `_variables.scss` に貼り付けてプロジェクトのデザイントークンとして使えるため、デザインシステムを構築するときの出発点として重宝する。

### 2. Coolors

**URL**: https://coolors.co/
**用途**: ランダム生成 + 手動調整でパレットを作る

スペースキーを押すたびにランダムなパレットが生成される。「なんとなくこんな雰囲気」という感覚から始めて、気に入った色だけロックして残りを再生成する使い方が直感的でわかりやすい。

生成したパレットはURLで共有でき、チームメンバーに共有するのも簡単だ。

### 3. Adobe Color

**URL**: https://color.adobe.com/ja/
**URL（トレンド）**: https://color.adobe.com/ja/trends
**用途**: 配色ルール（類似・補色・分割補色等）に基づいたパレット生成 + トレンドカラー

色相環の上でカラーを視覚的に操作しながら、配色ルールを確認できる。「補色モード」にして1色を動かすと、もう1色が自動的に補色の位置に移動するので、理論的に正しい配色を直感的に作れる。

Adobeのデザインコミュニティが投稿したパレットの閲覧・検索機能もあり、業種・ムード別のカラートレンドを把握するのにも役立つ。

### 4. Paletton

**URL**: https://paletton.com/
**用途**: 色相環ベースの配色計画ツール

色相環の上で直接ポイントを動かしてパレットを作る。モノカラー / 隣接2色 / 三角3色 / テトラッド4色の4パターンを切り替えながら、実際のWebページ風のプレビューで確認できる。

色彩理論を学びながら配色を決めたい人に向いている。

### 5. Accessible Colors（コントラスト確認）

**URL**: https://accessible-colors.com/
**用途**: テキスト・背景色のコントラスト比確認（WCAG準拠チェック）

WCAG 2.1（Web Content Accessibility Guidelines）では、テキストと背景のコントラスト比が **4.5:1以上**（通常テキスト）を求めている。

「このグレーの文字、読めるのかな？」という疑問をWCAGの基準で数値として確認できる。特にメインカラーを文字色に使う場合は必ずここで確認する。

### 6. CSS box-shadow ジェネレーター — ToolShare Lab

**URL**: [https://and-and.net/](https://and-and.net/)
**用途**: 影の色も含めたbox-shadowの生成

影の色にも配色は関係する。単純な黒（`rgba(0,0,0,0.x)`）ではなく、背景色と統一感のある色相の影を使うとより洗練した見た目になる。

[ToolShare Lab](https://and-and.net/) のbox-shadowジェネレーターは影の色をカラーピッカーで指定できるため、「メインカラーの暗めの色相で影を作る」といった細かい調整も視覚的に確認しながらできる。

### 7. UI Colors

**URL**: https://uicolors.app/create
**用途**: Tailwind CSS互換のカラースケール生成

1色を入力すると、Tailwind CSS の `50` から `950` まで（11段階）のカラースケールを自動生成する。Tailwindを使っていない場合でも、明度違いのカラースケールを作るのに便利だ。

```css
/* 生成例（Tailwind形式） */
--blue-50:  #eff6ff;
--blue-100: #dbeafe;
--blue-200: #bfdbfe;
...
--blue-900: #1e3a8a;
--blue-950: #172554;
```

### 8. Muzli Colors

**URL**: https://colors.muz.li/
**用途**: カラーの実際のUI使用例を確認

カラーコードを入力すると、そのカラーが実際のUI（ボタン・カード・ヘッダー等）でどう見えるかをモックアップで確認できる。

「コードではわかっても、実際に画面に乗せると印象が変わる」という問題を解消するツールで、配色を決める前の最終確認に使っている。

### 9. Color Hunt

**URL**: https://colorhunt.co/
**用途**: トレンドパレットのインスピレーション収集

デザイナーが投稿した4色パレットのコレクション。「ナチュラル」「ダーク」「パステル」などタグでフィルタリングでき、いいねが多いパレットから順に表示される。

「今どんな配色が流行っているか」を把握するのに向いている。インスピレーション探しの出発点として、週1回程度チェックしている。

## 配色ワークフローの実例

実際の案件でどの順番でツールを使うかを示す。

### ステップ1: イメージを言語化する

クライアントから「清潔感があって信頼できる感じ」「活発でエネルギッシュ」などのキーワードを引き出す。

### ステップ2: ベースカラーを決める

Color Hunt や Adobe Color のトレンドを参考に「青系 / 緑系」など色相の方向性を決める。

### ステップ3: パレットを展開する

[ToolShare Lab のカラーパレット生成ツール](https://and-and.net/) にベースカラーを入力し、補色・類似色のバリエーションを一括生成する。Coolors で追加の候補も生成する。

### ステップ4: コントラストを確認する

Accessible Colors で文字色と背景色の組み合わせを確認し、WCAG基準を満たしているか確認する。

### ステップ5: UIに乗せて確認する

Muzli Colors や実際のモックアップで、決定したパレットが実際のUI上でどう見えるかを確認する。

### ステップ6: CSS変数として定義する

[ToolShare Lab](https://and-and.net/) で生成したCSS変数形式のコードを、デザイントークンとしてプロジェクトに組み込む。

## まとめ

配色の迷いは「何かツールを1つ使えば解決」ではなく、複数のツールを用途に応じて組み合わせることで解決する。

- **出発点**: Color Hunt / Adobe Color でインスピレーション
- **パレット生成**: [ToolShare Lab](https://and-and.net/) / Coolors
- **理論的な整合性**: Paletton / Adobe Color
- **アクセシビリティ確認**: Accessible Colors
- **実装前の最終確認**: Muzli Colors
- **CSS出力**: [ToolShare Lab](https://and-and.net/) のCSS変数形式

このフローを習慣にすると、「なんとなくしっくりこない」という感覚的な悩みから抜け出せる。
